Estratégia de Implantação

Cenário 02: Sistema de Retaguarda "LancheRápido"

Estratégia Escolhida: Blue-Green Deployment

Equipe: [Heryvelton Douglas]

1. O Problema e Impacto

  • O LancheRápido opera com alto volume de processamento de pedidos em tempo real.
  • O cenário exige uma modificação severa: atualização da API do backend + migração estrutural de banco de dados.

A Dor Principal: Se a versão antiga e a nova dividirem conexões aleatórias simultaneamente, haverá colisão de estados e inevitável corrupção dos dados dos pedidos ativos.

2. A Estratégia Escolhida

Nossa equipe optou pelo Blue-Green Deployment.


  • Isolamento (Tudo ou Nada): O ambiente novo (Green) ficará 100% isolado da produção atual (Blue) até a validação completa.
  • Viabilidade: A empresa possui orçamento em nuvem para duplicar a infraestrutura temporariamente.
  • Risco Zero de Concorrência: Garante a integridade dos dados, pois não há roteamento fracionado (como no Canary).

3. Plano de Implementação

  • Fase Azul: 100% dos usuários utilizam o ambiente antigo via Load Balancer.
  • Provisionamento Verde: Instanciamos a nova API isoladamente e executamos a migração de dados à sombra.
  • O Chaveamento: Após validação, alteramos o apontamento do Load Balancer para redirecionar o tráfego 100% para o ambiente Verde em um único passo lógico.

Critério de Sucesso: Monitoramento ativo de estabilidade (ausência de erros HTTP 5xx) nos primeiros 5 minutos após a troca.

4. Plano de Contingência (Rollback)

Mecanismo de emergência estruturado para agir em menos de 10 segundos:


  • O ambiente antigo (Blue) será mantido ligado e intacto em segundo plano.
  • Se o monitoramento detectar falhas no ambiente novo (Green), scripts automatizados acionam o Rollback.
  • O Load Balancer é imediatamente reconfigurado para apontar o tráfego de volta para o ambiente Blue.
